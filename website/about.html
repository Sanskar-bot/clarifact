About ClariFact

Information moves faster
than verification.

AI has made fabrication cheap and distribution instant. ClariFact is a small, honest attempt to make verification a little easier — without pretending it can solve the problem alone.

What ClariFact does

Paste a claim. ClariFact searches the web, reads the top results in full, scores each source for credibility and relevance, then passes the evidence to an AI that returns a structured verdict: Supported, Contradicted, or Inconclusive — with the reasoning visible.

The verdict is not the end of the process — it's the start. Every result shows you exactly which sources were read and what they said. You're expected to check those sources yourself. The tool handles the retrieval; the judgement stays with you.

The full four-step pipeline is explained on the homepage.

Privacy — specific and explicit

Nothing is stored. The backend is fully stateless. When you submit a claim, the server processes it, returns a result, and discards the data. There is no database. No logs are kept of what you searched. No user accounts exist because there's no user data to attach an account to.

Only the claim text is sent to the server. If you're using the website checker, nothing about your browser, device, or browsing history is transmitted. If you're using the extension, only the text you've explicitly selected and chosen to check is sent — not the rest of the page, not your browsing history.

The AI does not learn from your queries. Requests to Nvidia's Nemotron model are inference-only — no fine-tuning or training happens from submitted claims.

Don't take our word for it — the backend source code is open: github.com/Sanskar-bot/clarifact.

Who built it

ClariFact is a solo, independent open-source project. It isn't a company, doesn't have investors, and isn't trying to grow into a business. It was built because the problem felt real and the tooling to address it has become genuinely accessible.

Being solo and open source is a trust signal, not a disclaimer — it means the code is readable, the project has no business incentive to collect data or sell access, and any issues can be raised and fixed publicly.

Source code, issues, and contributions: GitHub.
