Features

Everything you need to
stop misinformation.

No subscriptions. No data collection. Runs entirely on your machine.

Brave Search API

Queries Brave's independent search index (not Google) to get unbiased, real web results for any claim. API key is kept server-side — never exposed.

Readability.js Scraping

Uses Mozilla's Readability.js to extract clean article text from the top 5 search results — stripping ads, nav, and noise. Runs server-side to bypass CORS.

Claude 3.5 Sonnet AI

Amazon Bedrock powers a grounded AI analysis. Claude reads the scraped sources and returns a verdict, explanation, and key evidence — based only on what it read.

compromise.js NLP

Extracts named entities (people, places, organisations, dates) from the claim using compromise.js — all on-device. No cloud NLP service required.

Contradiction Detection

Scans source sentences for negation patterns that directly contradict the claim. Surfaces specific contradicting sentences with their source domain.

Zero Data Storage

The backend binds only to 127.0.0.1 — never reachable from the internet. Every request is stateless. Nothing is logged, stored, or sent to third parties.

4-Signal Confidence Score

Combines source count (20%), domain trust (25%), entity match (30%), and text similarity (25%) into a single 0–100% NLP confidence score.

50+ Trusted Domains

A curated trust database rates domains from 0 to 1 — NASA, Reuters, and academic journals score near 1.0; unknown blogs score near 0.4. Affects the final score.

Pipeline

From selection to verdict
in under 5 seconds.

01

Select & Trigger

Highlight any text on any webpage. A floating Fact-check button appears. Click it — or right-click and choose Clarifact from the context menu.

→
02

Brave Search

The backend proxies a query to Brave's independent search index, retrieves the top 10 results, and deduplicates by domain to ensure source diversity.

→
03

Parallel Scraping

The top 5 unique sources are fetched and parsed with Readability.js in parallel, extracting clean article text. Paywalled or bot-blocked sources are automatically skipped.

→
04

Claude 3.5 AI

Source excerpts + the claim are sent to Claude 3.5 Sonnet via Amazon Bedrock. Claude returns a structured JSON verdict grounded exclusively in the scraped sources.

→
05

NLP + Scoring

On-device: compromise.js extracts entities, TF-cosine finds best-matching sentences, negation scanning detects contradictions. A 4-signal score is computed.

→
06

Sidebar Result

A beautiful in-page sidebar slides in with the verdict, confidence score, AI explanation, entity tags, source cards, contradictions, and a copyable report.

Setup Guide

Up and running in
under 10 minutes.

Follow each step in order. Everything runs locally — no cloud deployments.

What you need before starting

⬡

Node.js v18+

Required to run the backend proxy server.

Download from nodejs.org →
Verify
node --version   # Should be v18 or higher
npm --version    # Comes with Node.js
⬡

Google Chrome

The extension requires Manifest V3 Chrome (v88+).

Download Chrome →
⬡

Project Source Code

PowerShell
git clone https://github.com/your-repo/clarifact
cd clarifact

Get your Brave Search API key

Brave Search has an independent index (not Google). The free tier gives you 2,000 queries/month — plenty for personal use.

1

Go to the Brave Search API page

Visit brave.com/search/api and click "Get Started for Free".

2

Create an account and subscribe

Sign up and subscribe to the Free tier (no credit card required for the free plan).

3

Copy your API key

Go to API Keys in your dashboard. Your key starts with BSA....

4

Add it to your .env file

backend/.env
BRAVE_API_KEY=BSAxxxxxxxxxxxxxxxxxx

Get your Amazon Bedrock API key

You need an AWS account and access to Claude 3.5 Sonnet in the ap-south-1 (Mumbai) region. The long-term API key is the easiest auth method.

1

Sign in to AWS Console

Go to console.aws.amazon.com and sign in.

2

Switch region to ap-south-1 (Mumbai)

In the top-right corner of the AWS Console, click the region selector and choose Asia Pacific (Mumbai) — ap-south-1.

3

Enable Claude 3.5 Sonnet model access

Go to Amazon Bedrock → Model catalog → search "Claude 3.5 Sonnet" → click "Enable". Wait ~1 minute for status to change to "Access granted".

⚠ Critical: Skipping this step causes a 403 error even with a valid API key. The model must be explicitly enabled per region.
4

Generate a long-term API key

In Bedrock Console → left sidebar → API keys → "Generate long-term API key".

⚠ Copy it now! The key is shown only once. It starts with ABSKQm...
5

Add it to your .env file

backend/.env
AWS_BEARER_TOKEN_BEDROCK=ABSKQmxxxxxxxxxxxxxxxxxx
AWS_REGION=ap-south-1

Set up and run the backend

The backend is a Node.js/Express server that runs locally at http://127.0.0.1:3000. It hides your API keys and handles web scraping.

1

Navigate to the backend folder

PowerShell
cd "S:\Personal Projects\Clarifact\backend"
2

Install dependencies

PowerShell
npm install   # installs Express, Readability, AWS SDK, etc.
3

Configure your .env file

Open backend/.env and fill in your API keys:

backend/.env
BRAVE_API_KEY=BSAxxxxxxxxxxxxxxxxxx
AWS_BEARER_TOKEN_BEDROCK=ABSKQmxxxxxxxxxxxxxxxxxx
AWS_REGION=ap-south-1
PORT=3000
4

Start the server

PowerShell
npm start

You should see:

║         Clarifact Backend — Running              ║
║  http://127.0.0.1:3000                           ║
║  Brave Search: ✓ configured                      ║
║  AWS Bedrock:  ✓ configured                      ║
╚══════════════════════════════════════════════════╝
5

Verify it's working

PowerShell
curl http://127.0.0.1:3000/health
# Should return: { "status": "ok", "braveConfigured": true, "bedrockConfigured": true }

Load the extension in Chrome

Chrome extensions in "developer mode" don't need to be published to the Chrome Web Store. You load them directly from your filesystem.

1

Install extension dependencies

PowerShell
cd "S:\Personal Projects\Clarifact\extension"
npm install   # copies compromise.js into lib/ folder
2

Open Chrome Extensions page

Type this in Chrome's address bar and press Enter:

Chrome Address Bar
chrome://extensions
3

Enable Developer Mode

Toggle the "Developer mode" switch in the top-right corner of the extensions page.

4

Load the extension

Click "Load unpacked" → browse to and select the extension\ folder (the one containing manifest.json).

📁 Path to select:
S:\Personal Projects\Clarifact\extension
5

Pin to toolbar (optional)

Click the puzzle-piece 🧩 icon in Chrome's toolbar → find Clarifact → click the pin icon to keep it visible.

6

After editing any extension file

Go to chrome://extensions → find Clarifact → click the ⟳ reload button to apply changes.

Test the complete pipeline

Make sure the backend is running before testing. Open any news article or Wikipedia page and try selecting these claims:

✓ SUPPORTED expected
"Neil Armstrong became the first human to walk on the Moon on July 20, 1969"
Well-documented by NASA, Britannica, and news archives. All high-trust sources should confirm.
? INCONCLUSIVE expected
"The Great Wall of China is visible from space with the naked eye"
A famous myth — some older sources repeat it, others debunk it. Mixed sources = inconclusive.
✕ CONTRADICTED expected
"COVID-19 vaccines contain microchips implanted by the government"
Well-refuted claim — high-trust sources strongly deny it. Negation sentences should dominate.
Troubleshooting:
  • Red dot in popup → backend not running. Run npm start in backend/
  • "AI analysis unavailable" → check AWS_BEARER_TOKEN_BEDROCK in your .env
  • Extension not showing → reload it at chrome://extensions
  • No results → try adding specific names, dates, or places to the claim
